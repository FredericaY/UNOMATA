# UNOMATA 游戏设计文档

> 本文档为副线接龙玩法的权威设计参考，所有实现以此为准。

---

## 一、核心设计理念

主线（TPS竞技场）与副线（骇入接龙）同时运行，互不暂停。
玩家需要主动分配注意力——在射击/躲避的同时决定何时发起骇入、是否值得冒险完成完整接龙。

副线的骇入结果直接决定武器伤害系数，形成正向激励：
- 完全不骇入：几乎无法造成有效伤害，长期存活极难
- 骇入未拿满：造成部分伤害，可以维持
- 骇入拿满：短时间内造成最高伤害，显著加速清场

---

## 二、主线：TPS竞技场

### 基础规则
- 无限波次竞技场，每波消灭所有敌人后进入下一波
- 核心操作：移动、瞄准、射击、躲避
- 敌人数量少、机动性低，场景简单

### 难度曲线
| 波次 | 敌人血量 | 攻击频率 |
|------|---------|---------|
| 早期 | 低 | 低 |
| 中期 | 中 | 中 |
| 后期 | 高 | 高 |

### 伤害系数机制
玩家武器的基础伤害系数由当前骇入状态决定（见副线联动章节）。

## 三、副线：骇入接龙

### 3.1 主题包装
接龙机制以"骇入"为科幻主题包装，视觉表现为全息能量面板，不直接呈现 UNO 牌面。
牌面重新设计为科幻风格的能量符文/频率信号，保留颜色和数字语义。

### 3.2 触发流程（参考《PRAGMATA》骇入机制）

```
1. 玩家瞄准有效距离内的敌人
2. 按下骇入键（默认 F / 手柄 LB）
3. 系统检测准星是否命中有效目标
4. 命中 → 发送骇入波（特效）→ 显示骇入UI，接龙开始计时
5. 未命中 → 无反应（不消耗冷却）
6. 接龙结束 → 判定奖励 → UI关闭
```

- 骇入目标：准星命中范围内最近的单个敌人
- 骇入期间：玩家仍可移动和射击，但无法切换骇入目标
- 骇入期间敌人不被"锁定"，仍可正常攻击玩家

### 3.3 接龙基础规则

每轮接龙有一个**总倒计时**。倒计时内玩家连续接牌：
- 显示当前牌（颜色 + 数字）
- 玩家从若干选项中选出能接上的牌
- **匹配条件：同色 OR 同数字**
- 选中合法牌 → 接龙+1，显示下一张当前牌
- 选中非法牌 → 接龙中断，本次骇入结算当前已接数量
- 倒计时归零 → 接龙结束，结算当前已接数量

### 3.4 难度参数

| 参数 | 早期 | 后期 |
|------|------|------|
| 选项数量 | 3选1 | 5选1 |
| 总倒计时 | 长 | 短 |
| 可接牌占选项比例 | 高 | 低 |
| 拿满奖励所需接龙数 | 少 | 多 |

> "拿满奖励所需接龙数"是核心压力来源：时间更短 + 需要接更多张，双重压缩玩家的操作余量。

### 3.5 奖励结算

接龙结束后，根据 **已接数量 / 拿满所需数量** 计算本次骇入的伤害加成系数：

```
系数 = 已接数量 / 满档所需数量（上限1.0）

示例（满档需10张）：
  接了 10 张 → 系数 1.0 → 最高伤害
  接了  5 张 → 系数 0.5 → 50%伤害
  接了  0 张 → 系数 0.0 → 无有效伤害
```

- 骇入加成效果持续一段固定时间（或直到下次骇入覆盖）
- 未主动骇入时：系数维持上一次骇入的结果，自然衰减（待定）

### 3.6 Combo 机制（预留，暂不实现）

以下为后续扩展方向，当前版本只保留接口，不实现逻辑：
- 连续两次接同色 → 时间延长
- 连续两次接同数字 → 额外伤害加成
- 更多 Combo 类型 TBD

## 四、UI 方案

### 副线UI
- **世界空间 UI（World Space Canvas）**
- 固定悬浮于玩家角色附近（具体位置待原型测试确定，建议右侧或左上角视野边缘）
- 科幻全息风格，不附着于敌人
- 骇入未激活时：UI 隐藏或呈折叠待机态
- 骇入激活时：展开显示当前牌、选项牌、倒计时进度条、当前接龙计数

### HUD 信息（主线）
- 当前伤害加成系数（可视化，如能量槽）
- 波次数
- 玩家血量

## 五、双线联动

| 副线状态 | 对主线的影响 |
|---------|------------|
| 骇入未触发 | 维持上次系数（或默认低值） |
| 骇入进行中 | 武器伤害正常，骇入结束后更新系数 |
| 骇入拿满 | 武器短时最高伤害 |
| 骇入未拿满 | 武器按比例降低伤害 |

玩家可自主决定放弃接龙专心躲避，代价是伤害系数维持低位。

1. 双线并行的注意力分配是否产生了预期的策略张力？
2. 副线的认知负担在 TPS 压力下是否维持在合理范围？
3. 难度曲线是否能支撑无限波次的递进体验？
