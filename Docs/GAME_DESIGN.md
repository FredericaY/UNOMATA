# UNOMATA 游戏设计文档

> 本文档为副线接龙玩法的权威设计参考，所有实现以此为准。

---

## 一、核心设计理念

主线（TPS竞技场）与副线（骇入接龙）同时运行，互不暂停。
玩家需要主动分配注意力——在射击/躲避的同时决定何时发起骇入、骇入哪个目标、是否值得冒险完成完整接龙。

骇入的本质是**降低目标敌人的伤害减免**，而非强化玩家自身武器。效果只作用于被骇入的那个敌人：
- 未被骇入的敌人：拥有极高伤害减免（约95%），几乎无法打出有效伤害
- 骇入中/骇入结算有效期内的敌人：伤害减免按接龙结果比例降低，满档时无减免甚至附加负减免（受到伤害加成）
- 骇入满后继续接：溢出部分转化为生命回复技能充能

这一机制带来的核心决策：**先骇入谁？** 面对多个敌人时，玩家需要判断集中骇入最危险的目标，还是分散骇入快速清场。

---

## 二、主线：TPS竞技场

### 基础规则
- 无限波次竞技场，每波消灭所有敌人后进入下一波
- 核心操作：移动、瞄准、射击、躲避
- 敌人数量少、机动性低，场景简单

### 难度曲线
| 波次 | 敌人血量 | 攻击频率 |
|------|---------|---------|
| 早期 | 低 | 低 |
| 中期 | 中 | 中 |
| 后期 | 高 | 高 |

### 伤害减免机制
敌人自带基础伤害减免，骇入是降低该减免的唯一手段（见副线联动章节）。

## 三、副线：骇入接龙

### 3.1 主题包装
接龙机制以"骇入"为科幻主题包装，视觉表现为全息能量面板，不直接呈现 UNO 牌面。
牌面重新设计为科幻风格的能量符文/频率信号，保留颜色和数字语义。

### 3.2 触发流程（参考《PRAGMATA》骇入机制）

```
1. 玩家瞄准有效距离内的敌人
2. 按下骇入键（默认 F / 手柄 LB）
3. 系统检测准星是否命中有效目标
4. 命中 → 发送骇入波（特效）→ 显示骇入UI，接龙开始计时
5. 未命中 → 无反应（不消耗冷却）
6. 接龙结束 → 判定奖励 → UI关闭
```

- 骇入目标：准星命中范围内最近的单个敌人
- 骇入期间：玩家仍可移动和射击，但无法切换骇入目标
- 骇入期间敌人不被"锁定"，仍可正常攻击玩家

### 3.3 接龙基础规则

每轮接龙有一个**总倒计时**。倒计时内玩家连续接牌：
- 显示当前牌（颜色 + 数字）
- 玩家从若干选项中选出能接上的牌
- **匹配条件：同色 OR 同数字**
- 选中合法牌 → 接龙+1，显示下一张当前牌
- 选中非法牌 → **立刻扣玩家血量，骇入中断，结算当前已接数量**
- 倒计时归零 → 接龙结束，结算当前已接数量（不扣血）

### 3.4 难度参数

| 参数 | 早期 | 后期 |
|------|------|------|
| 选项数量 | 3选1 | 5选1 |
| 总倒计时 | 长 | 短 |
| 可接牌占选项比例 | 高 | 低 |
| 拿满奖励所需接龙数 | 少 | 多 |

> "拿满奖励所需接龙数"是核心压力来源：时间更短 + 需要接更多张，双重压缩玩家的操作余量。

### 3.5 奖励结算

接龙结束后，根据 **已接数量 / 拿满所需数量** 计算对**该目标敌人**的伤害减免削减系数：

```
减免削减 = min(已接数量 / 满档所需数量, 1.0)

示例（满档需10张，敌人基础减免95%）：
  接了 10 张 → 减免削减 1.0 → 敌人减免归零，甚至获得额外受击加成
  接了  5 张 → 减免削减 0.5 → 敌人减免降至约47%
  接了  0 张 → 减免削减 0.0 → 敌人保持95%减免，几乎无法有效输出
```

- **效果只作用于被骇入的目标敌人**，其他敌人不受影响
- 骇入加成效果持续一段固定时间后**清零**（不衰减，直接归零）
- 下次骇入结算时用新系数覆盖，每次骇入独立计算，不累积

### 3.6 溢出奖励：生命回复技能充能

达成满档（系数1.0）后仍在倒计时内继续接龙，**溢出的接牌数**转化为生命回复技能的充能点数：

```
溢出充能 = max(已接数量 - 满档所需数量, 0)
```

- 生命回复技能可以缓存若干次充能（具体缓存上限待平衡调整）
- 玩家主动触发生命回复技能时消耗充能，恢复一定血量
- 溢出充能不影响武器伤害系数（系数上限始终为1.0）
- 设计意图：给熟练玩家提供持续接龙的额外收益，形成高风险高回报的技术表达空间

### 3.6 Combo 机制（预留，暂不实现）

以下为后续扩展方向，当前版本只保留接口，不实现逻辑：
- 连续两次接同色 → 时间延长
- 连续两次接同数字 → 额外伤害加成
- 更多 Combo 类型 TBD

## 四、UI 方案

### 副线UI
- **世界空间 UI（World Space Canvas）**
- 固定悬浮于玩家角色附近（具体位置待原型测试确定，建议右侧或左上角视野边缘）
- 科幻全息风格，不附着于敌人
- 骇入未激活时：UI 隐藏或呈折叠待机态
- 骇入激活时：展开显示当前牌、选项牌、倒计时进度条、当前接龙计数

### HUD 信息（主线）
- 当前伤害加成系数（可视化，如能量槽）
- 波次数
- 玩家血量

## 五、双线联动

骇入效果**绑定在目标敌人身上**，不是全局状态。每个敌人各自维护自己的骇入状态。

| 敌人骇入状态 | 对该敌人的影响 |
|------------|--------------|
| 未被骇入 / 骇入效果已过期 | 保持基础减免（约95%），几乎无法有效输出 |
| 骇入进行中 | 维持上次骇入系数，骇入结算后更新 |
| 骇入结算：未拿满 | 减免按比例削减，持续固定时间后清零 |
| 骇入结算：拿满 | 减免完全清除，甚至附加受击加成，持续固定时间后清零 |
| 接错牌触发 | 玩家扣血，骇入中断结算，目标敌人获得当前已接数量的减免削减 |
| 溢出接龙 | 玩家生命回复技能充能+1（不影响目标敌人的减免系数，上限已为0） |

**核心战术决策点**：
- 先骇入哪个敌人？（优先威胁最大的还是最容易击杀的？）
- 当前骇入值不值得完成？（敌人快死了，骇入收益变低）
- TPS压力过大时是否中断骇入专心躲避？

## 六、验证目标

项目结束后需要能回答：
1. 双线并行的注意力分配是否产生了预期的策略张力？
2. 副线的认知负担在 TPS 压力下是否维持在合理范围？
3. 难度曲线是否能支撑无限波次的递进体验？
